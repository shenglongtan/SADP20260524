# Compare.py 功能分析与使用说明

## 1. 文件简介

`Compare.py` 是由 `Compare.ipynb` 转换而来，用于对多个实验结果进行性能对比可视化。该脚本读取多个结果文件中的评估指标数据，生成基于 `seaborn` 的箱线图，比较不同模型或参数配置在不同预测步长上的表现。

## 2. 主要功能

- 读取多个结果目录下的 `.pkl` 数据文件
- 从数据中提取指定指标（如 `R^2` 和 `MAE`）
- 将数据合并成统一表格
- 将宽表转换为长表，方便 `seaborn` 绘图
- 绘制每个指标的两个子图：
  - 左图：按预测步长对比不同模型的分布
  - 右图：按模型对比整体误差分布
- 支持自定义绘图风格与字体设置

## 3. 脚本结构

### 3.1 导入依赖

- `os`：处理文件路径
- `pandas`：读取和处理表格数据
- `seaborn`：绘制统计图表
- `matplotlib.pyplot`：绘图控制和显示
- `matplotlib.gridspec.GridSpec`：控制子图宽度比例

### 3.2 配置参数

- `EXPORT_PATH`：数据文件的基础路径，默认空字符串表示直接使用绝对路径。
- `DPI`：图像分辨率，默认为 `300`。
- `IMG_TYPE`：导出图片类型，默认 `.tif`。
- `E_NAME`：需要提取和展示的指标列表，目前为 `['R^2', 'MAE']`。

### 3.3 数据路径与图例配置

- `folder_name`：多个实验结果的目录名称列表。
- `abs_folder`：将基础结果目录与文件夹名拼接，形成 `.pkl` 文件基础路径。
- `legend_l`：与 `folder_name` 一一对应的图例标签列表。

### 3.4 数据加载与处理

1. 初始化空 `DataFrame` `plt_df`。
2. 遍历 `abs_folder`，读取对应的 `.pkl` 文件。
3. 筛选 `Type` 列中属于 `E_NAME` 的数据。
4. 添加 `Model` 列，标记当前结果来源。
5. 拼接到 `plt_df`。
6. 重置索引并打印头部数据。

### 3.5 数据转换为绘图格式

- 使用 `pd.melt()` 将宽格式数据转换为长格式，生成 `plt_df_long`。
- 保留列：`Run_Id`, `Model`, `Type`。
- 新生成列：`Horizon Step`, `Value`。

### 3.6 绘图逻辑

- 使用 `matplotlib` 设置全局字体和数学公式样式。
- 使用 `GridSpec` 生成 `2 x len(E_NAME)` 的子图布局。
- 左侧子图：按 `Horizon Step` 绘制箱线图；
- 右侧子图：按 `Model` 绘制总体误差箱线图；
- 统一隐藏多余轴标签并添加参考虚线。
- 将图例统一放在图像底部。

## 4. 使用说明

### 4.1 运行前准备

- 确保 Python 环境已安装：
  - `pandas`
  - `seaborn`
  - `matplotlib`
- 确保 `E:\\DataForCode\2_STGNNOutput` 目录存在，并包含如下结构：
  - `20260120_163808\Result.pkl`
  - `20260120_185839\Result.pkl`
- 结果文件应包含列：`Run_Id`, `Type`, 以及不同预测步长对应的数值列。

### 4.2 基本运行方式

```python
python Compare.py
```

### 4.3 常用修改点

- 修改 `folder_name`：加入更多实验结果目录。
- 修改 `legend_l`：调整图例名称，使其与不同实验一一对应。
- 修改 `E_NAME`：若要展示其他指标，如 `['MAPE', 'RMSE']`，直接替换即可。
- 如果需要将图像保存到 `EXPORT_PATH`，可在脚本末尾增加 `fig.savefig(...)` 逻辑。

## 5. 适用场景

- 对比不同参数配置下模型预测精度差异。
- 展示多种模型在不同预测步长上的稳定性。
- 为论文或报告生成可视化结果。

## 6. 注意事项

- `EXPORT_PATH` 目前为空字符串，若希望读取相对路径文件，可根据当前脚本目录修改。
- `plt_df_long` 的 `Horizon Step` 需要是数值或可排序的标签，否则箱线图顺序可能不一致。
- `legend_l` 的数量必须与 `folder_name` 保持一致，否则会出现标签错位。

---

## 7. 建议

- 若希望对结果文件进行批量自动发现，可用 `os.listdir()` 代替手工写 `folder_name` 列表。
- 若希望支持更多指标展示，可把 `E_NAME` 设为外部配置参数并添加命令行参数解析。
- 若结果文件命名格式统一，可把 `.pkl` 路径构造逻辑封装成函数，便于复用。
